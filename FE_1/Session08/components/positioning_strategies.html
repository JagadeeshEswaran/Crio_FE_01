Positioning Strategies

Strategy 1 - Static
  • This is default for all the element in HTML.
  • It means that the elements will be placed "normally"(i.e)
    • Vertically stacking for block &
    • Horizontal for in-line elements.
  • Attributes like left, right, top, bottom & z-index will have no effect with "Static"

Strategy 2 - Absolute
  • Allows us to place an element exactly where we want!
  • We can us attributes to set the Location weherever we want.
  • These values are relative to the next (or Closest) parent element's positioning
  • An absolute element is removed from the flow of the page.
  • (i.e) These elements won't affect other elements.
  • If no position attribute in parent, then by default the element is place relative to the page itself.
  • Caution! We should be careful of not to overuse it.

Strategy 3 - Relative
  • Relative means -> "Relative to its original position on the page"
  • In essence, if we set position: relative but no attributes, then it will have NO effect on positioning.
  • If we set a position attributes to some lenght -> element will shift from its original position.
  • Super Power Added
    • position: relative, absolute & fixed adds one more ability to an element.
    • It adds the "z-index" property.
    • This is the ability to be stacked over other elements.
    • Even is no z-index is given also, the element appears on top of "static" positioned elements.

Strategy 4 - Fixed
  • With

    position: fiex

    the element is positioned relative to the viewport or browser window.
  • A fixed element stays where it was when the content is scrolled.
  • This is useful for navigation bars & headers.
